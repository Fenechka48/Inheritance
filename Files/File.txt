<div class="dwhat" colspan="2" id="CH_i1238760">
 The "Bible" of Windows API programming:<br>
<br>
"Programming Windows", Fifth Edition by Charles Petzold.<br>
<a href="https://www.amazon.com/Programming-Windows®-Fifth-Developer-Reference/dp/157231995X/">https://www.amazon.com/Programming-Windows®-Fifth-Developer-Reference/dp/157231995X/</a><br>
<br>
The book was originally written and published for Win9X, so not every code example will compile without some modifications.<br>
<br>
The code examples have been updated and put into a gitub repo.<br>
<br>
<a href="https://github.com/recombinant/petzold-pw5e">https://github.com/recombinant/petzold-pw5e</a><br>
<br>
There is a 6th edition of "Programming Windows," but it uses C# and is targeted for Win 8.<br>
<br>
There a similar book for MFC programming: "Programming Windows with MFC", Second Edition by Jeff Prosise.<br>
<a href="https://www.amazon.com/gp/product/1572316950/">https://www.amazon.com/gp/product/1572316950/</a><br>
<br>
As I said, neither book is recent, but they still do a good job of showing the nitty-gritty of bashing together code for low-level Win programming.<br>
<br>
There really is no "one stop shopping" online when it comes to tutorials and example code for learning Win32 technology anymore. Examples are flung far and wide in tidbits here and there.<br>
<br>
theForger's Win32 API Tutorial is still available. It is a bare-bones beginners look at the Win32 API.<br>
<br>
<a href="http://www.winprog.org/tutorial/">http://www.winprog.org/tutorial/</a><br>
<br>
Fair warning: the code was written back in Win9X days, the Win32 API has changed since then. I have a github repo where I updated the code to better reflect modern Win API coding practices.<br>
<br>
<a href="https://github.com/GeorgePimpleton/theForger-winapi-tutorial">https://github.com/GeorgePimpleton/theForger-winapi-tutorial</a><br>
<br>
Two more WinAPI tutorials.<br>
<a href="https://riptutorial.com/winapi">https://riptutorial.com/winapi</a><br>
<a href="http://www.flounder.com/mvp_tips.htm">http://www.flounder.com/mvp_tips.htm</a><br>
<br>
A tutorial for MFC, though I can't say it if it good or not. I've only skimmed the contents.<br>
<a href="https://www.tutorialspoint.com/mfc/index.htm">https://www.tutorialspoint.com/mfc/index.htm</a><br>
<br>
Older versions of the WinAPI SDK were available for download, and included example sample code along with the Win SDK documentation accessible offline. I believe it is still possible to download the archives, though I haven't looked for any for years.<br>
<br>
MS has git repositories for Windows API sample code online, though the samples are coded to use the WinRT and C++. The original WinAPI was C based.<br>
<br>
MS documentation for the WinAPI is all online now. Hard to follow and get an overview of what's available IMO.<br>
<br>
There's a lot of 3rd party code available out on the interwebz for C-based WinAPI and MFC still.<br>
<br>
Microsoft as a github repo of a lot of "classic" code examples for Windows Desktop apps.<br>
<a href="https://github.com/microsoft/Windows-classic-samples">https://github.com/microsoft/Windows-classic-samples</a><br>

<div class="dwhat" colspan="2" id="CH_i1238762">
Do you mean a Windows 'app' (ie as in UWP - Universal Windows Program) or a C++ program that uses a 3rd party library for its gui? There is a big difference!<br>
<br>
What George describes above is for Microsoft WIN32 API (basically using c-style functions) and Microsoft MFC (using C++ classes).<br>
<br>
If you want a Windows UWP app, then there is C++/Winrt.<br>
<a href="https://learn.microsoft.com/en-us/windows/uwp/cpp-and-winrt-apis/intro-to-using-cpp-with-winrt">https://learn.microsoft.com/en-us/windows/uwp/cpp-and-winrt-apis/intro-to-using-cpp-with-winrt</a><br>
<br>
But before you get into buying books/resources etc for C++ gui programming, you need to decide upon which 3rd party library/framework etc you are going to use. The books are usually specific to a particular library.<br>
<br>
For a sample of those available, see Graphical User Interface in<br>
<a href="https://en.cppreference.com/w/cpp/links/libs">https://en.cppreference.com/w/cpp/links/libs</a><br>
<br>
PS. Stroustrup's book 'Programming: Principles and Practice Using C++' uses the FLTK gui library.<br>
 <br>
PPS. There's also C++/cli which uses .net. Although if you go down the .net route IMO you'd be better off using c# rather than C++.<br>
<br>
WIN32, MFC, cli, winrt (and c# and .net) are all part of Visual Studio (although depending upon your installation options may need to be installed from within VS Installer). So they cannot be used with another compiler.<br>
<br>
However, there is Embarcadero C++ Builder which has its VCL (Visual Component Library). There is the free Community Edition:<br>
<a href="https://www.embarcadero.com/products/cbuilder/starter">https://www.embarcadero.com/products/cbuilder/starter</a><br>
<br>
This may be more what you're looking for.<br>
<br>
Other 3rd party libraries (eg FLTK) if used will need to be downloaded/installed etc separately.<br>

<div class="dwhat" colspan="2" id="CH_i1238774">
Great that you have ideas you want to develop.  Yet you dismiss out of hand suggestions to assist you from people who have experience.<br>
<br>
A Windows GUI app can be created by one of several different ways, using different available technologies.  Technologies that now are no longer shackled to just Windows-based devices.<br>
<br>
There are apps that are targeted to be what is known as Desktop apps, or apps that are universal apps that can execute on different Windows-based platforms.<br>
<br>
Windows app development has a very diverse ecosystem.  Visual Studio is a decent integrated development environment with available packages for the different technologies.<br>
<br>
What you can learn about Windows development is available online, though not together in one neat package like a book.  A book is usually out-dated from the moment its published and gets more so the older the book.  Online resources can be less out-dated.  Online MS resources are hit and miss in that area.<br>
<br>
Writing apps for Windows - Windows apps | Microsoft Learn<br>
<a href="https://learn.microsoft.com/en-us/windows/apps/get-started/">https://learn.microsoft.com/en-us/windows/apps/get-started/</a><br>
<br>
<table class="quote"><tr><td class="qd">If your app will only run on Windows, we recommend using the Windows App SDK and WinUI 3.</td></tr></table><br>
Hmmmmm, Windows App SDK.  Follow that link and we get:<br>
<a href="https://learn.microsoft.com/en-us/windows/apps/windows-app-sdk/">https://learn.microsoft.com/en-us/windows/apps/windows-app-sdk/</a><br>
<br>
<table class="quote"><tr><td class="qd">The Windows App SDK doesn't replace the Windows SDK or existing desktop Windows app types such as .NET (including Windows Forms and WPF) and <b><i><u>desktop Win32 with C++.</u></i></b> (emphasis added) Instead, the Windows App SDK complements these existing tools and app types with a common set of APIs that developers can rely on across these platforms.</td></tr></table><br>
Desktop Win32 app development is not 'dying', no matter what you think or have been told.<br>
<br>
Maybe if you gave a brief description of what you'd like to create, so we can better focus on pointing out the relevant technologies best suited for doing that.<br>
<br>
I certainly wouldn't think of trying to create an Android, iOS, Mac or Linux app using Desktop Win32 tech.
